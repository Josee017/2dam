0. Concepción y Diseño Inicial

Idea del Proyecto: El objetivo fue crear una "Suite de Archivos del Futuro". Quería alejarme de los exploradores de archivos aburridos y crear una plataforma web segura, con estética Cyberpunk/Glassmorphism, que permitiera visualizar y editar código o PDFs sin necesidad de descargarlos.

1. Fase de Diseño (Inspiración)

Descripción:

"Lo primero que hice fue usar Gemini para generar ideas visuales de cómo quería que fuera la app. Quería un modo futurista y intenso, con ese modo oscuro y los colores neón. Estos diseños me sirvieron de base para saber qué pedirle después al asistente de programación."

2. Fase de Desarrollo (Evolución de la UI)

Prompt / Acción realizada:

"Le pedí al asistente que creara un formulario de login que 'flotara' sobre un fondo animado de partículas. La clave aquí fue la validación estricta: si el correo no terminaba en .com o .es, el borde se iluminaba en rojo neón. Quería que la seguridad se sintiera visual."

Dashboard Principal (Resumen de Actividad)

Prompt / Acción realizada:

"Esta fue la parte central. Necesitaba una zona de 'Drag & Drop' que reaccionara. Implementamos una animación de escaneo láser que se activa al soltar un archivo, dando esa sensación de 'Procesando Datos' antes de mostrarlos en la lista."

Vista de Perfil y Ajustes

Prompt / Acción realizada:

"No quería un menú de configuración aburrido. Diseñamos un 'Theme Engine' visual con tarjetas de colores (Cyan, Purple, Lime). Además, añadí un slider interactivo que controla la intensidad del desenfoque (blur) de los cristales en tiempo real, permitiendo al usuario personalizar su nivel de inmersión."

Arreglo y mejora (Revision)

Aqui hay varios prompts que use para la mejora y implementacion de varias secciones y arreglos de las que no iban.

 
Prompt 1: Debugging Crítico (Lógica de Contraseñas)"Actúa como un Desarrollador Senior de JavaScript (Debugger). Tenemos un bug crítico en la Web App: el formulario de 'Cambiar Contraseña' permite modificar la clave sin validar la anterior.
Tu Misión: Reescribir SOLAMENTE la lógica JS de updatePassword(). Asegúrate de comparar el input con la contraseña guardada en el estado global (currentUser) y lanza un error bloqueante si no coinciden."
Prompt 2: Modernización de UX (Adiós Alert)"Actúa como un Diseñador UX/UI. Vamos a eliminar por completo los alert() nativos del navegador, ya que rompen la estética futurista.
Implementación: Crea un sistema de Notificaciones Flotantes (Toasts) estilo Neon/Glassmorphism. Deben deslizarse desde la derecha con bordes de color semántico (Verde para éxito, Rojo para error) y autodestruirse tras 4 segundos."
Prompt 3: Seguridad y Persistencia"Actúa como un Ingeniero de Seguridad Frontend. Necesitamos realismo en el módulo de autenticación.
1. Regex Estricto: El email debe contener obligatoriamente '@' y terminar en '.com' o '.es'.
2. Medidor de Fuerza: Añade una barra visual que cambie de color (Rojo/Naranja/Verde) según la complejidad de la contraseña.
3. Simulación DB: Usa localStorage para persistir un array de usuarios registrados y validar credenciales reales al hacer login."

Nota sobre el desarrollo: Este proyecto evolucionó de un simple boceto visual a una SPA (Single Page Application) compleja y funcional. La iteración constante con la IA permitió refinar detalles como el modo Claro/Oscuro y la simulación de base de datos local, logrando un producto que se siente profesional y futurista.
